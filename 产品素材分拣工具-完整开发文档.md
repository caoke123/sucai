## 1. 项目概述

### 1.1 这个工具做什么

用户本地有大量产品图片，需要将它们按照**妙手 ERP** 规定的文件夹结构整理好，再交给 PIM 中台系统进行 SKU 编码和 Excel 导出。

**本工具负责：**
- 打开本地文件夹，展示所有图片缩略图
- 用户点击标签（主图 / SKU图 / 详情图 / 尺寸图 / 证书），再点选图片完成分类标注
- 录入基础产品信息（标题、类目、属性、SKU 规格组合）
- 点击"生成素材包"，自动创建规范目录结构、重命名并复制图片、输出 `product.json`

**本工具不负责：**
- SKU 编码（由 PIM 中台完成）
- 售价 / 库存录入（由 PIM 中台完成）
- 生成 Excel 导入模板（由 PIM 中台完成）

### 1.2 输出的素材包结构

```
[产品名称]_素材包/
├── 产品主图/
│   ├── 主图_1.jpg
│   ├── 主图_2.jpg
│   └── 主图_3.jpg
├── SKU图/
│   ├── 白色_1.jpg
│   └── 粉色_2.jpg
├── 详情图/
│   ├── 详情图_1.jpg
│   └── 详情图_2.jpg
├── 尺寸图表/
│   └── 尺寸图表.png
├── 产品证书/
│   └── 产品证书_1.pdf
├── 产品视频/          （空文件夹，占位）
└── product.json       （结构化数据，供 PIM 中台读取）
```

## 2. 技术栈说明

### 为什么选这套技术栈

| 技术 | 版本 | 用途 | 选择原因 |
|---|---|---|---|
| **Electron** | 33.x | 桌面应用壳 | 允许访问本地文件系统，跨平台（Mac/Win） |
| **React** | 18.x | 前端 UI 框架 | 与 PIM 中台同技术栈，组件可直接复用 |
| **TypeScript** | 5.x | 编程语言 | 强类型，减少 bug，代码提示友好 |
| **Vite** | 5.x | 构建工具 | 开发热更新极快 |
| **electron-vite** | 2.x | Electron+Vite 整合脚手架 | 官方推荐方案，开箱即用 |
| **Tailwind CSS** | 3.x | 样式框架 | 与 PIM 中台保持一致 |
| **Zustand** | 4.x | 状态管理 | 轻量，代码量少，无需 Redux 复杂配置 |
| **sharp** | 0.33.x | 图片处理 | 生成缩略图，只跑在 Node.js 主进程 |
| **Express** | 4.x | HTTP 服务 | 嵌入 PIM 模式时的本地 Agent 服务 |

### 运行环境要求

- **Node.js** >= 20.0.0（推荐 LTS 版本）
- **npm** >= 10.0.0
- **操作系统**：macOS 12+ 或 Windows 10+

## 5. 数据结构定义

所有类型定义集中在 `src/shared/types.ts`，前后端共用。**请先创建此文件，后续所有模块都依赖它。**

```typescript
// src/shared/types.ts

// 图片可以被标注的类别
export type ImageLabel =
  | '主图'
  | 'SKU图'
  | '详情图'
  | '尺寸图'
  | '证书'
  | '未分类';

// 每张图片的数据结构
export interface ImageFile {
  id: string;                    // 唯一 ID，如 "img_0"
  originalPath: string;          // 原始文件的完整路径
  thumbnailDataUrl: string;      // base64 缩略图，格式："data:image/jpeg;base64,..."
  fileName: string;              // 文件名，如 "DSC_0001.jpg"
  fileExt: string;               // 扩展名，如 ".jpg"
  label: ImageLabel;             // 当前标注的类别，默认 "未分类"
  skuSpec?: string;              // SKU图专用：对应的规格值，如 "白色"
  order?: number;                // 同类图片中的排序编号（生成时自动赋值）
}

// SKU 规格行（一个规格组合）
export interface SkuSpecRow {
  id: string;                    // 唯一 ID
  spec1?: string;                // 规格1的值，如 "白色"
  spec2?: string;                // 规格2的值，如 "36"
}

// 产品基础信息（本工具录入，不含售价/库存，由 PIM 补充）
export interface ProductInfo {
  title: string;                 // 产品标题（必填）
  currency: string;              // 货币类型，默认 "CNY"
  sourceUrl: string;             // 货源链接
  sourcePlatform: string;        // 货源平台，如 "1688"
  productNo: string;             // 产品主编号
  category: string;              // 货源类目
  description: string;           // 详情描述
  attributes: string;            // 属性，用中文分号分隔
  spec1Name: string;             // 规格1的名称，如 "颜色"
  spec2Name: string;             // 规格2的名称，如 "尺码"
  skuSpecs: SkuSpecRow[];        // SKU 规格列表
}

// 整理完成后的输出 JSON（product.json）
export interface ProductOutput {
  title: string;
  currency: string;
  sourceUrl: string;
  sourcePlatform: string;
  productNo: string;
  category: string;
  description: string;
  attributes: string;
  spec1Name: string;
  spec2Name: string;
  images: {
    main: string[];              // 主图文件名列表
    sku: string[];               // SKU图文件名列表
    detail: string[];            // 详情图文件名列表
    size: string[];              // 尺寸图文件名列表
    certificate: string[];       // 证书文件名列表
  };
  skuSpecs: Array<{
    spec1?: string;
    spec2?: string;
  }>;
  createdAt: string;             // ISO 时间字符串
  toolVersion: string;           // 工具版本号
}

// IPC 通信：扫描文件夹的请求参数
export interface ScanFolderRequest {
  folderPath: string;

// IPC 通信：扫描文件夹的返回结果

  error?: string;

// IPC 通信：整理文件的返回结果
export interface OrganizeResult {
  success: boolean;
  outputPath?: string;           // 生成的素材包完整路径
  error?: string;
```

### 6.4 文件夹扫描模块

```typescript
// src/main/ipc/scanFolder.ts
import { ipcMain } from 'electron'
import fs from 'fs'
import path from 'path'
import sharp from 'sharp'
import type { ScanFolderResult, ImageFile } from '../../shared/types'

// 支持的图片格式
const IMAGE_EXTENSIONS = new Set(['.jpg', '.jpeg', '.png', '.webp', '.gif', '.bmp', '.tiff'])

export function registerScanFolderHandler() {
  ipcMain.handle('scan-folder', async (_event, folderPath: string): Promise<ScanFolderResult> => {
    try {
      // 读取文件夹内所有文件
      const allFiles = fs.readdirSync(folderPath)
      const imageFiles = allFiles.filter(fileName => {
        const ext = path.extname(fileName).toLowerCase()
        return IMAGE_EXTENSIONS.has(ext)
      })

      if (imageFiles.length === 0) {
        return { success: true, images: [] }
      }

      // 并发生成所有图片的缩略图（最多 5 张同时处理，避免内存溢出）
      const results: ImageFile[] = []
      const chunkSize = 5

      for (let i = 0; i < imageFiles.length; i += chunkSize) {
        const chunk = imageFiles.slice(i, i + chunkSize)
        const chunkResults = await Promise.all(
          chunk.map(async (fileName, chunkIndex) => {
            const filePath = path.join(folderPath, fileName)
            const ext = path.extname(fileName).toLowerCase()

            try {
              // 生成 200x200 缩略图，JPEG 格式，质量 75
              const thumbBuffer = await sharp(filePath)
                .resize(200, 200, { fit: 'cover', position: 'centre' })
                .jpeg({ quality: 75 })
                .toBuffer()

              const imageFile: ImageFile = {
                id: `img_${i + chunkIndex}`,
                originalPath: filePath,
                thumbnailDataUrl: `data:image/jpeg;base64,${thumbBuffer.toString('base64')}`,
                fileName,
                fileExt: ext,
                label: '未分类',
              }
              return imageFile
            } catch {
              // 单张图片处理失败时跳过，不中断整体流程
              console.warn(`跳过无法处理的文件: ${fileName}`)
              return null
            }
          })
        )
        results.push(...chunkResults.filter(Boolean) as ImageFile[])
      }

      return { success: true, images: results }
    } catch (error) {
      return {
        success: false,
        error: `扫描文件夹失败: ${(error as Error).message}`
      }
    }
  })
}
```

### 6.5 文件整理模块

```typescript
// src/main/ipc/organizeFiles.ts
import { ipcMain } from 'electron'
import fs from 'fs'
import path from 'path'
import type { OrganizeRequest, OrganizeResult, ProductOutput, ImageFile } from '../../shared/types'

// 标签 → 文件夹名称的映射
const LABEL_TO_FOLDER: Record<string, string> = {
  '主图':   '产品主图',
  'SKU图':  'SKU图',
  '详情图': '详情图',
  '尺寸图': '尺寸图表',
  '证书':   '产品证书',
}

// 标签 → product.json 中 images 字段的 key
const LABEL_TO_KEY: Record<string, keyof ProductOutput['images']> = {
  '主图':   'main',
  'SKU图':  'sku',
  '详情图': 'detail',
  '尺寸图': 'size',
  '证书':   'certificate',
}

export function registerOrganizeFilesHandler() {
  ipcMain.handle('organize-files', async (_event, payload: OrganizeRequest): Promise<OrganizeResult> => {
    try {
      const { outputFolderPath, images, productInfo } = payload

      // 用产品标题（去除特殊字符）作为素材包文件夹名
      const safeName = productInfo.title
        .replace(/[\\/:*?"<>|]/g, '_')  // 替换不合法的文件名字符
        .substring(0, 50)               // 最长50字符
        .trim()
      const packageName = `${safeName}_素材包`
      const packagePath = path.join(outputFolderPath, packageName)

      // 如果目标文件夹已存在，先删除（覆盖重新生成）
      if (fs.existsSync(packagePath)) {
        fs.rmSync(packagePath, { recursive: true, force: true })
      }

      // 创建所有子文件夹（包括空的产品视频文件夹）
      const subFolders = ['产品主图', 'SKU图', '详情图', '尺寸图表', '产品证书', '产品视频']
      for (const folder of subFolders) {
        fs.mkdirSync(path.join(packagePath, folder), { recursive: true })
      }

      // 按标签分组图片，并分配排序编号
      const groupedImages: Record<string, ImageFile[]> = {}
      for (const image of images) {
        if (image.label === '未分类') continue  // 未分类的图片不处理
        if (!groupedImages[image.label]) groupedImages[image.label] = []
        groupedImages[image.label].push(image)
      }

      // 复制并重命名文件，同时记录最终文件名
      const outputImages: ProductOutput['images'] = {
        main: [], sku: [], detail: [], size: [], certificate: []
      }

      for (const [label, labelImages] of Object.entries(groupedImages)) {
        const folderName = LABEL_TO_FOLDER[label]
        const imageKey = LABEL_TO_KEY[label]
        if (!folderName || !imageKey) continue

        const destDir = path.join(packagePath, folderName)

        labelImages.forEach((image, index) => {
          const count = index + 1
          let newFileName: string

          if (label === 'SKU图' && image.skuSpec) {
            // SKU图命名规则：规格名_序号.扩展名，如 白色_1.jpg
            newFileName = `${image.skuSpec}_${count}${image.fileExt}`
          } else {
            // 其他图片命名规则：类型_序号.扩展名，如 主图_1.jpg
            const labelName = label.replace('图', '')
            newFileName = `${labelName}_${count}${image.fileExt}`
          }

          // 复制文件
          fs.copyFileSync(image.originalPath, path.join(destDir, newFileName))
          outputImages[imageKey].push(newFileName)
        })
      }

      // 生成 product.json
      const productOutput: ProductOutput = {
        title: productInfo.title,
        currency: productInfo.currency || 'CNY',
        sourceUrl: productInfo.sourceUrl || '',
        sourcePlatform: productInfo.sourcePlatform || '',
        productNo: productInfo.productNo || '',
        category: productInfo.category || '',
        description: productInfo.description || '',
        attributes: productInfo.attributes || '',
        spec1Name: productInfo.spec1Name || '',
        spec2Name: productInfo.spec2Name || '',
        images: outputImages,
        skuSpecs: productInfo.skuSpecs.map(s => ({
          spec1: s.spec1,
          spec2: s.spec2,
        })),
        createdAt: new Date().toISOString(),
        toolVersion: '1.0.0',
      }

      const jsonPath = path.join(packagePath, 'product.json')
      fs.writeFileSync(jsonPath, JSON.stringify(productOutput, null, 2), 'utf-8')

      return { success: true, outputPath: packagePath }
    } catch (error) {
      return {
        success: false,
        error: `生成素材包失败: ${(error as Error).message}`
      }
    }
  })
}
```

```typescript
import type { ScanFolderResult, OrganizeRequest, OrganizeResult } from '../../shared/types'

const isElectron = typeof window !== 'undefined' && !!window.electronAPI

// PIM 嵌入模式时，本地 Agent 服务的地址
const AGENT_BASE_URL = 'http://localhost:18899'

export function useFileSystem() {

  // 选择文件夹
  const selectDirectory = async (): Promise<string | null> => {
    if (isElectron) {
      return window.electronAPI.selectDirectory()
    }
    // PIM 模式：弹出自定义输入框让用户输入路径（或由 Agent 弹窗）
    const path = window.prompt('请输入产品图片文件夹的完整路径：')
    return path || null
  }

  // 扫描文件夹
  const scanFolder = async (folderPath: string): Promise<ScanFolderResult> => {
    if (isElectron) {
      return window.electronAPI.scanFolder(folderPath)
    }
    const res = await fetch(`${AGENT_BASE_URL}/scan-folder`, {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ folderPath }),
    })
    return res.json()
  }

  // 整理文件
  const organizeFiles = async (payload: OrganizeRequest): Promise<OrganizeResult> => {
    if (isElectron) {
      return window.electronAPI.organizeFiles(payload)
    }
    const res = await fetch(`${AGENT_BASE_URL}/organize-files`, {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify(payload),
    })
    return res.json()
  }

  return { selectDirectory, scanFolder, organizeFiles }
}
```

```typescript
// src/renderer/store/useSorterStore.ts
import { create } from 'zustand'
import { immer } from 'zustand/middleware/immer'
import type { ImageFile, ImageLabel, ProductInfo, SkuSpecRow } from '../../shared/types'

type Step = 'folder' | 'labeling' | 'info' | 'preview' | 'done'

interface SorterStore {
  // 步骤控制
  currentStep: Step
  setStep: (step: Step) => void

  // 文件夹路径
  sourceFolderPath: string
  outputFolderPath: string
  setSourceFolderPath: (path: string) => void
  setOutputFolderPath: (path: string) => void

  // 图片列表
  images: ImageFile[]
  setImages: (images: ImageFile[]) => void
  setImageLabel: (id: string, label: ImageLabel, skuSpec?: string) => void
  setMultipleLabels: (ids: string[], label: ImageLabel) => void

  // 当前激活的标签（标注模式下）
  activeLabel: ImageLabel
  setActiveLabel: (label: ImageLabel) => void

  // 已选中待标注的图片 ID
  selectedImageIds: Set<string>
  toggleImageSelected: (id: string) => void
  clearSelection: () => void
  selectAll: () => void

  // 产品信息
  productInfo: ProductInfo
  setProductInfo: (info: Partial<ProductInfo>) => void
  addSkuSpec: () => void
  removeSkuSpec: (id: string) => void
  updateSkuSpec: (id: string, field: 'spec1' | 'spec2', value: string) => void

  // 输出结果
  outputPath: string
  setOutputPath: (path: string) => void

  // 重置所有状态
  reset: () => void
}

const defaultProductInfo: ProductInfo = {
  title: '',
  currency: 'CNY',
  sourceUrl: '',
  sourcePlatform: '1688',
  productNo: '',
  category: '',
  description: '',
  attributes: '',
  spec1Name: '颜色',
  spec2Name: '尺码',
  skuSpecs: [],
}

export const useSorterStore = create<SorterStore>()(
  immer((set) => ({
    currentStep: 'folder',
    setStep: (step) => set(state => { state.currentStep = step }),

    sourceFolderPath: '',
    outputFolderPath: '',
    setSourceFolderPath: (path) => set(state => { state.sourceFolderPath = path }),
    setOutputFolderPath: (path) => set(state => { state.outputFolderPath = path }),

    images: [],
    setImages: (images) => set(state => { state.images = images }),
    setImageLabel: (id, label, skuSpec) => set(state => {
      const img = state.images.find(i => i.id === id)
      if (img) {
        img.label = label
        if (skuSpec !== undefined) img.skuSpec = skuSpec
      }
    }),
    setMultipleLabels: (ids, label) => set(state => {
      const idSet = new Set(ids)
      state.images.forEach(img => {
        if (idSet.has(img.id)) img.label = label
      })
    }),

    activeLabel: '主图',
    setActiveLabel: (label) => set(state => {
      state.activeLabel = label
      state.selectedImageIds = new Set()
    }),

    selectedImageIds: new Set(),
    toggleImageSelected: (id) => set(state => {
      if (state.selectedImageIds.has(id)) {
        state.selectedImageIds.delete(id)
      } else {
        state.selectedImageIds.add(id)
      }
    }),
    clearSelection: () => set(state => { state.selectedImageIds = new Set() }),
    selectAll: () => set(state => {
      state.selectedImageIds = new Set(state.images.map(i => i.id))
    }),

    productInfo: { ...defaultProductInfo },
    setProductInfo: (info) => set(state => {
      Object.assign(state.productInfo, info)
    }),
    addSkuSpec: () => set(state => {
      state.productInfo.skuSpecs.push({
        id: `sku_${Date.now()}`,
        spec1: '',
        spec2: '',
      })
    }),
    removeSkuSpec: (id) => set(state => {
      state.productInfo.skuSpecs = state.productInfo.skuSpecs.filter(s => s.id !== id)
    }),
    updateSkuSpec: (id, field, value) => set(state => {
      const spec = state.productInfo.skuSpecs.find(s => s.id === id)
      if (spec) spec[field] = value
    }),

    outputPath: '',
    setOutputPath: (path) => set(state => { state.outputPath = path }),

    reset: () => set(state => {
      state.currentStep = 'folder'
      state.sourceFolderPath = ''
      state.outputFolderPath = ''
      state.images = []
      state.selectedImageIds = new Set()
      state.activeLabel = '主图'
      state.productInfo = { ...defaultProductInfo }
      state.outputPath = ''
    }),
  }))
)
```

## 9. 两种运行模式

### 模式一：Electron 独立桌面应用

直接运行，无需额外配置。所有文件操作通过 IPC 在主进程完成。

```
用户界面 (React)
    ↓ window.electronAPI.scanFolder()
Preload (IPC Bridge)
    ↓ ipcRenderer.invoke('scan-folder')
主进程 (Node.js)
    ↓ fs.readdirSync / sharp
本地文件系统
```

### 模式二：嵌入 PIM 中台

需要在本地额外运行一个轻量 Express 服务（`agent`），作为浏览器与文件系统之间的桥梁。

```typescript
// agent/index.ts
import express from 'express'
import cors from 'cors'
import { scanRoute } from './routes/scan'
import { organizeRoute } from './routes/organize'

const app = express()
app.use(cors({ origin: '*' }))        // 允许 PIM 中台跨域访问
app.use(express.json({ limit: '50mb' }))

app.use('/scan-folder', scanRoute)
app.use('/organize-files', organizeRoute)

const PORT = 18899
app.listen(PORT, '127.0.0.1', () => {
  console.log(`产品素材分拣 Agent 已启动：http://localhost:${PORT}`)
})
```

```typescript
// agent/routes/scan.ts
import { Router } from 'express'
import fs from 'fs'
import path from 'path'
import sharp from 'sharp'

const router = Router()
const IMAGE_EXTENSIONS = new Set(['.jpg', '.jpeg', '.png', '.webp', '.gif', '.bmp'])

router.post('/', async (req, res) => {
  const { folderPath } = req.body
  try {
    const files = fs.readdirSync(folderPath)
      .filter(f => IMAGE_EXTENSIONS.has(path.extname(f).toLowerCase()))

    const images = await Promise.all(files.map(async (fileName, index) => {
      const filePath = path.join(folderPath, fileName)
      const thumbBuffer = await sharp(filePath)
        .resize(200, 200, { fit: 'cover' })
        .jpeg({ quality: 75 })
        .toBuffer()
      return {
        id: `img_${index}`,
        originalPath: filePath,
        thumbnailDataUrl: `data:image/jpeg;base64,${thumbBuffer.toString('base64')}`,
        fileName,
        fileExt: path.extname(fileName).toLowerCase(),
        label: '未分类',
      }
    }))

    res.json({ success: true, images })
  } catch (error) {
    res.json({ success: false, error: (error as Error).message })
  }
})

export { router as scanRoute }
```

```json
// electron-builder 配置中添加

  "npmRebuild": true,
  "buildDependenciesFromSource": false
```

### Q5：如何在同一台电脑上同时运行 PIM 和 Agent

确保 Agent 端口（18899）未被占用。PIM 中台和 Agent 均为本地服务，不需要联网。
